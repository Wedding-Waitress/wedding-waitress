

## Plan: Sync PDF Download Seat Layout with Page Display

### Problem
The PDF download (generated by `individualTableChartEngine.ts`) does not match the page display (`IndividualTableChartPreview.tsx`). Specifically:
1. Seat numbers are not centered inside the round seat circles in the PDF
2. Guest name positions around seats don't match the page display

### Root Cause
The preview uses CSS flexbox (`display: flex; align-items: center; justify-content: center`) for centering seat numbers inside circles, which renders perfectly in browsers. The engine uses `text-align: center; line-height: 44px`, which `html2canvas` does not render reliably.

Additionally, there are minor positioning value differences between preview and engine.

### Changes (single file only)

**File: `src/lib/individualTableChartEngine.ts`**

1. **Fix seat circle centering**: Replace `text-align: center; line-height: 44px` with `display: flex; align-items: center; justify-content: center` to match the preview exactly. This is the same method the preview uses and will render correctly in `html2canvas`.

2. **Align guest name gap values with preview**: The preview uses percentage-based gaps that translate to specific pixel values. Ensure the engine's pixel gaps match exactly:
   - `circleGapX`: keep at 26px (matches 5.2% of 500px)
   - `circleGapY`: adjust to match the preview's gap plus the extra PDF-only offset the user previously requested (keep the larger gap for top seat as requested earlier, but ensure it produces the correct visual)

3. **Match name label transform values**: Ensure all `transform` values for guest name labels match the preview's approach (`translate(-50%, -50%)` for center, `translate(-100%, -50%)` for right, `translate(0, -50%)` for left).

No changes to `IndividualTableChartPreview.tsx` or any other file.

### Technical Detail

The core fix in the engine's seat circle HTML (around line 1164-1182):

```
// FROM:
text-align: center;
line-height: 44px;
overflow: hidden;

// TO:
display: flex;
align-items: center;
justify-content: center;
```

This single change ensures `html2canvas` properly centers the seat numbers, matching the preview's flexbox approach exactly.
